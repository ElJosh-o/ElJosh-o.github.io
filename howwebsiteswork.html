Task 1: How Websites Work

Websites have two main components:
1.The Front End/Client Side – This is how your browser renders a website
2. Back End/Server Side – A server that processes requests and returns responses
At it’s most basic a client makes a request to a server and the server responds with information

Task 2: HTML

Websites are primarily created using three tools
HTML – To build a site and define it’s structure
CSS- Cascading Style Sheets. These provide style that can be spread across the whole site
JavaScript – A scripting language that allows for implementation of complex features on webpages.

The HTML structure above has the following comp onents. - <#!DOCTYPE html> defines the page as a HTML5 document. This tells the browser to use HTML5 to interpret the page and helps with standardisation across the different browsers. DOCTYPEs are required for legacy reasons - <#html> is the root element of the page. Everything else comes after (within) the html tags - <#head> contains information about the page title. e.g what shows up on the title of the browser. - <#body> defines the HTML document’s body; only content inside the body is shown in the browser. - <#h1> defines a large heading. There are six or so different tags. - <#p> is for a paragraph In addition to the above there are tags for buttons, images and a lot more. Elements can have class tags e.g

Task 5: HTML Injection

Injection, a classic.. This is when unfiltered user input impacts on what a website is doing. Ideally anywhere that a user can input text will have ways of sanitising it to ensure that no code can be run. HTML injection is client side, effecting only how the website outputs content to the client.
